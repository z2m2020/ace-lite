# ACE-Lite Engine v0.3.90

Release date: 2026-04-17

## What Changed

- Long-term memory capture/reporting now uses a shared layering metadata surface, including stable abstraction, support, freshness, contradiction, and last-confirmed fields.
- Feedback observability is now aggregated across developer issues, issue reports, and selection feedback instead of treating developer feedback as the only visible signal.
- `runtime status` and `runtime doctor` now expose a unified `observation_overview` block so operators can see cross-store gaps directly.
- `plan_quick` now more aggressively demotes obvious ACE-Lite-generated tooling artifacts for code-intent queries, reducing retrieval pollution from local task/feedback markdown.
- Runtime feedback scoping is tighter: when a repo filter is not passed explicitly, runtime summaries now infer repo scope from the latest runtime match and then the requested root.
- Default `~/.ace-lite/...` feedback paths now resolve through the user home correctly instead of being treated as repo-relative paths.
