# ACE-Lite Engine v0.3.92

Release date: 2026-04-18

## What Changed

- `ace_dev_feedback_summary` now accepts an explicit `root` through the MCP surface, and the unified feedback overview exposes `scope_diagnostics` so operators can see which issue store path and root source were actually used.
- CLI feedback commands now stay root-consistent: `feedback record` and `feedback stats` both resolve relative `profile-path` values under the requested `--root`, preventing silent read/write drift across repositories.
- Selection feedback stats now expose path-level explainability fields such as `matching_event_count`, `matched_terms`, `avg_position`, `capture_rate`, `sample_queries`, and `sample_events`.
- MCP tool metadata now locks `root` exposure for `ace_feedback_record`, `ace_feedback_stats`, and `ace_dev_feedback_summary`, reducing the chance that service support and tool schema drift apart.
- Canonical repo identity detection now handles real git worktree `.git` indirection layouts, so fresh `ace-lite-mcp --self-test --root <worktree>` runs keep `default_repo` on the stable repository name instead of the worktree directory name.
